Featured project · Case study

Azure E-Commerce Lakehouse

Live on GitHub

A production-style data platform for an e-commerce business: streaming clickstream and batch order data land in a Delta lakehouse, get transformed with PySpark, and surface in Power BI — all provisioned as code and deployed through CI/CD.

The problem

E-commerce data arrives two ways at once — a live stream of clickstream events and nightly batches of orders from an operational SQL database. Analytics teams need both, joined, historized, and queryable — without paying for an always-on warehouse.

The solution

A medallion-architecture Delta lakehouse on ADLS Gen2: Event Hubs ingests the stream, Azure SQL feeds the batch path, Databricks handles PySpark transforms with SCD Type 2 history, and Synapse Serverless serves Power BI — pay-per-query, no warehouse to babysit.

Architecture
Streaming + batch in one platform

Real-time clickstream through Event Hubs alongside nightly batch loads from Azure SQL — unified in the same Delta tables.

SCD Type 2 history tracking

Dimension changes are versioned, not overwritten — analysts can query customer and product state as of any date.

Tested, versioned, automated

A test suite guards the transforms, Terraform pins the infrastructure, and GitHub Actions ships changes safely.
